# Предобработка и визуализация больших данных

## Структура

В директории [data](./data) в общих чертах показана структура файлов:
- [raw](./data/raw) - должны храниться исходные файлы (в данном случае файл *prices.csv*);
- [plots](./data/plots) - результаты визуализации, например, графики. Сейчас там лежат
    примеры после тестового запуска, подробнее ниже.

Также подразумевается наличие дополнительных директорий для хранения очищенных файлов и витрин
(Можно посмотреть [.gitignore](Lab-3/.gitignore) текущего уровня, где они добавлены).

Кроме файлов данных присутствуют и скрипты для выполнения логики [scripts](./scripts):
- [generate_data](./scripts/generate_data.py) - отвечает за создание набора данных в формате
    *CSV*. По умолчанию создается не менее 1М строк (из-за создания дубликатов их может оказаться больше).
    Генерируются случайные данные из настроек, включая ошибки, варианты написания, пропуски и т.д.;
- [clean_and_mart](./scripts/clean_and_mart.py) - очистка сырых данных и создание витрин.
    Для обработки используется **PySpark**. Производится удаление дубликатов, приведение типов и
    удаление строк с пропусками, если их нельзя исправить. Далее расчитываются три витрины и сохраняются
    в папку в виде *parquet*;
- [plot_mart](./scripts/plot_mart.py) - базовая визуализация с помощью **pandas** и **matplotlib**.
    Данные считываются из ранее сохраненных витрин, и строятся два графика с сохранением в
    [plots](./data/plots).

В коде в некоторых местах присутствуют комментарии для упрощения понимания происходящего.

В [compose](./compose.yaml) указаны сервисы для запуска соответствующих скриптов. Об этом далее.


## Запуск

Запуск производится через `Docker` последовательно:
1. ```docker compose run --rm data_gen``` - создает файл *price.csv* согласно настройкам в файле;
2. ```docker compose run --rm spark``` - запускает обработку данных с помосщью `Spark`;
3. ```docker compose run --rm viz``` - считывает результаты прошлого шага и производит визуализацию.

Желательно не менять константы в скриптах, но если это все же необходимо, то важно обеспечить
согласованность данных, например, директорий.
