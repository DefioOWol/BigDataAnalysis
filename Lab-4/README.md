# Apache Hadoop. Парадигма MapReduce. Apache Spark.

## Структура

В директории [data](./data) задается структура файлов:

- [raw](./data/raw) - исходные файлы после генерации:
  - *prices.csv* - структурированные факты о ценах (табличный *CSV*);
  - *products.json* - структурированный справочник товаров (по одному *JSON*-объекту на строку);
  - *reviews.json* - отзывы с **неструктурированным** полем `text` (остальные поля структурированы);
- `./data/clean` - очищенные данные в *parquet*. Не сохраняется в **git**;
- `./data/mart` - витрины в *parquet*. Не сохраняется в **git**;
- `./data/export` - выгрузки витрин в *JSON* и *CSV* для внешнего использования. Не сохраняется в **git**;
- [plots](./data/plots) - графики визуализации (примеры после запуска `viz`).

Общие константы и пути вынесены в [domain.py](./scripts/domain.py), чтобы не дублировать их между генератором и Spark.

Скрипты [scripts](./scripts):

- [generate_data](./scripts/generate_data.py) - формирует *CSV* и два *JSON* (справочник и отзывы).
    По умолчанию не менее 1M строк в ценах (из-за дубликатов может быть больше);
- [spark_job](./scripts/spark_job.py) - **Spark-приложение**: импорт из разных источников, очистка,
    расчет витрин, объединение справочника и метрик по тексту отзывов, запись *parquet* и экспорт в *JSON*/*CSV*;
- [plot_mart](./scripts/plot_mart.py) - визуализация витрин через **pandas** и **matplotlib** (чтение из *parquet*).

В [compose](./compose.yaml) заданы сервисы для последовательного запуска.


## Запуск

Запуск производится через `Docker` последовательно (аналогично [Lab-3](../Lab-3/README.md)):

1. `docker compose run --rm data_gen` - создает *prices.csv*, *products.json*, *reviews.json*;
2. `docker compose run --rm spark_job` - обработка в Spark;
3. `docker compose run --rm viz` - построение графиков в [plots](./data/plots).

Изменение доступных параметров (констант) согласуется в [domain.py](./scripts/domain.py).


## Что делает [Spark-job](./scripts/spark_job.py)

- **Импорт:** *CSV* (цены), *JSON* (товары), *JSON* (отзывы с текстом);
- **Очистка и витрины:** как и в исходной лабораторной - по категориям, магазинам, дневные изменения цен (окна).
    Дополнительно витрина `product_snapshot` (цены + бренд/срок годности из справочника + агрегаты по отзывам:
    число отзывов, средняя оценка, средняя длина текста и число слов);
- **Экспорт:** `daily_by_category` (*JSON*), `product_snapshot` (*CSV*),
    файл `import_manifest.txt` со списком путей импорта в директории `./data/export`.
