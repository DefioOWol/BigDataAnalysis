# Spark MLlib и Spark ML

Лабораторная продолжает [пайплайн](../Lab-4/README.md): те же источники
(*CSV* цен, *JSON* справочника, *JSON* отзывов с неструктурированным полем `text`),
та же очистка и витрины.
Дополнительно выполняется анализ с **Spark MLlib** и сохраняются результаты в [data/ml](./data/ml/).


## Структура

- [data/raw](./data/raw) - после шага `data_gen`: *prices.csv*, *products.json*, *reviews.json*;
- `data/clean`, `data/mart`, `data/export` - аналогично [ЛР 4](../Lab-4/README.md);
- [data/ml](./data/ml) - выход MLlib: *metrics.json*, `product_clusters` (*parquet*),
    `rating_predictions_sample` (*JSON*). В **git** сохраняются только метрики моделей в качестве примера
    (подробнее про ML ниже);
- [scripts](./scripts):
  - [generate_data](./scripts/generate_data.py) - генерация данных;
  - [spark_job](./scripts/spark_job.py) - Spark: ETL + витрины + **MLlib**;
  - [plot_mart](./scripts/plot_mart.py) - визуализация витрин.

В [compose](./compose.yaml) те же сервисы, что и в [ЛР 4](../Lab-4/README.md).


## Запуск

Последовательно:

1. `docker compose run --rm data_gen`;
2. `docker compose run --rm spark_job`;
3. `docker compose run --rm viz` (опционально, графики по витринам).

Параметр памяти драйвера: переменная окружения `SPARK_DRIVER_MEMORY` (по умолчанию в *compose* задано **2g**).


## MLlib

1. **Линейная регрессия** (`LinearRegression` в `Pipeline`).  
    Целевая переменная - `rating` отзыва.  
    Признаки: **TF-IDF** по неструктурированному тексту `text` (*Tokenizer* -> *HashingTF* -> *IDF*),
    индекс категории товара из справочника (*StringIndexer*), числовой признак `shelf_life_days`.  
    На тестовой выборке считаются **RMSE** и **R2**.

2. **Кластеризация K-means** (*k* = 5) по витрине `product_snapshot`.  
    Признаки: средняя цена, средний рейтинг, число отзывов.  
    Метрики: **silhouette**, **WSSSE** (внутрикластерная сумма квадратов).

Итоговые численные метрики - в [metrics.json](./data/ml/metrics.json).

```
Метрики в сохраненном файле отображают некорректные показатели, так как данные синтетические и связи между текстом отзыва и оценкой нет (они выбираются случайно).
Поэтому, например, показатель R2 близок к нулю.
```


## Сравнительный анализ ЛР 4 и ЛР 5

| Аспект | ЛР 4 | ЛР 5 |
|--------|------|------|
| Источники | *CSV* + два *JSON* (в т.ч. текст отзывов) | Аналогично |
| Фокус обработки | Очистка, окна, группировки, витрины, простые текстовые метрики (длина, число слов) | То же **плюс** обучение моделей MLlib |
| Результат | Parquet-витрины, экспорт *JSON*/*CSV*, графики по агрегатам | Витрины **и** модели: регрессия по тексту+справочнику, кластеры товаров |
| Интерпретация | Описательная аналитика (средние, динамика, снимок по товару) | Дополнительно прогноз/структурирование (рейтинг по признакам, сегменты товаров) |

ЛР 4 отвечает на вопросы "какие средние цены" и "как менялись". ЛР 5 на тех же данных строит
**предсказательную** и **кластерную** модель, то есть переходит от чистой агрегации
к методам машинного обучения из **Spark MLlib**.


## Краткое описание выполненных действий

Скопирована основная логика ЛР 4 (генерация данных, доменные константы, *compose*).  
В [spark_job.py](./scripts/spark_job.py) после расчета витрин вызывается блок `run_mllib`.
Из сырых *JSON* собирается датасет "отзыв + справочник товара", строится `pipeline`
с текстовым вектором TF-IDF и числовыми/категориальными признаками, после чего модель
регрессии обучается и оценивается на тестовой выборке. Затем из готовой витрины `product_snapshot`
формируются признаки для **K-means** и сохраняются кластеры. Метрики сериализуются в
[JSON](./data/ml/metrics.json).
